## 六、关键工作流（接手必复用）

### 完成单集分镜的标准SOP

```
第1步：阅读剧本对应集（来自01_原始剧本/）
第2步：进入"导演级深度思考"模式
   - 分析叙事定位
   - 设计情绪曲线
   - 标记爽点位置
   - 列出X大优化方向
第3步：按场次输出v2级分镜
   - 不少于剧本节拍数的镜头
   - 标注⭐核心爆点
   - 每镜含完整10字段（编号/时长/景别/角度/运镜/画面/光影/情绪/声音/质感词）
第4步：自评分（百分制）
第5步：附上"不改剧本"的合规检查表
第6步：询问用户是否推送GitHub
```

### 完成单集"逐镜直投提示词"的SOP

```
第1步：基于v2分镜脚本提取
第2步：每镜独立打包（不依赖上下文）
第3步：直接用"特写镜头一位30岁东亚男性..."自然语言开头
第4步：最后一段加入质感词（ARRI Alexa+柯达色调+8K等）
第5步：标注难度（简单/中等/难/极难）
第6步：附"首帧图垫底"清单
```

### 完成BGM Suno提示词的SOP

```
第1步：识别情绪段落（每3-8镜为一段）
第2步：按 Genre+BPM+Mood+Instruments+Vocal+Style 结构
第3步：英文输出（Suno英文效果远好于中文）
第4步：标注每段时长和切换时间点
第5步：附剪辑节奏表+卡点标注
```

## 七、AI助理工作风格守则

### ✅ 用户喜欢的方式
1. **先思考再行动**——明确要求时必须给"分析+规划"再执行
2. **结构化输出**——表格、列表、分级标签
3. **数据化结论**——给百分比、给评分、给镜头数
4. **诚实自评**——发现自己的错误不掩饰，立刻承认
5. **主动推GitHub**——重要资产做完主动询问推送

### ❌ 用户讨厌的方式
1. **拍马屁式吹嘘**——不要自夸"完美""最佳"
2. **重复废话**——不要在每个回复开头都重复"好的我来帮你"
3. **过度英文化**——除非工具需要，用中文
4. **隐藏问题**——发现剧本/分镜有矛盾要立刻指出，不能糊弄
5. **询问无意义选项**——要主动给建议，不要把决策推给用户
